# Game Vision Roadmap — Void Dominion

> Итог дизайн-совещания владельца и ассистента. **8 зафиксированных якорей видения**
> + **48 кирпичиков** в 6 фазах, каждый grounded в реальном коде (6-агентный
> воркфлоу-обзор дизайна + авторинг). Это карта «куда ведём игру», не жёсткий план
> сроков; порядок — по критическому пути.
>
> Формат: `ID · зона · вид · effort · deps` → **Готово, когда** · *зачем*.
> Виды: `fix-unify` (свести/починить расхождение) · `new-content` (данные + модуль) ·
> `new-system` (новая механика) · `polish`. Effort: S/M/L.
> Парные доки: `roadmap.md`, `backlog.md` (общий список), `gdd.md`, `resource-economy.md`,
> `squadrons-roadmap.md`, `missiles-roadmap.md`, `account-level.md`, `metagame.md`.

## 0. Восемь якорей видения (решения владельца)

1. **Темп:** оба режима — быстрый (`×24`, ~дни) И медленный (`×1–4`, недели) как выбираемые скорости.
2. **Масштаб:** «мета-социальный massive» — матчи ≤10 мест; «массовость» = слой корпораций/AvA/лиг НАД матчами. Персистентный мета-мир (Контур 2) — не сейчас. Карты — размера матча.
3. **Монетизация/прогрессия:** гибрид — XP/уровень аккаунта = ТОЛЬКО горизонталь (косметика/опции/контент, ноль стат-преимущества); ПЛЮС одна отдельная **непокупаемая** вертикальная ось = мастерство/престиж (напр. число сыгранных сессий) с престижем/титулами + **ограниченным, не копящимся** мелким edge (не «стена ветеранов»). **Выпилить +5% из мета-дерева.** Донат — только горизонталь.
4. **Глубина боя:** комбинированные рода войск — глубина через состав (авианосцы↔ПВО, арта, ракеты, дроны как сеть контр). НЕ пер-юнит RPS урона/брони. Эскадрильи/авианосцы промоутятся из прототипа в ядро.
5. **География:** плоский захват + террейн — без фронтов/снабжения/смежности/зон контроля. Гео значима только через террейн, родную землю, перехват на трассах.
6. **Фракции:** символические — косметика + мелкие плоские пассивы, одинаковый геймплей. Различие НЕ идёт от фракций.
7. **Тех-дерево:** качественные разблокировки — большинство узлов = новый юнит/здание/род/тактика (рост словаря эффектов), НЕ плоские %. Дерево — главный раздатчик контента; пейсит оба темпа.
8. **Первый мета-столп:** корп-социальный клуб — постоянный корп-чат + присутствие + медали/престиж (вне матча корпы механически пусты).

> **Каскад (принятое следствие):** раз фракции И география плоские, вся идентичность
> билда и стратегическая глубина концентрируются на **тех-дерево + состав сил +
> герои + лоадаут корабля** — эти четыре рычага обязаны быть богатыми.

## 1. Критический путь

```
Фаза 0 · Фундамент (гейт всего)
      → Фаза 1 · Бой (combined-arms)  ┐
      → Фаза 2 · Техи (качественные)  ┴─ параллельно (глубина/идентичность)
            → Фаза 3 · Ретеншн  ┐
            → Фаза 4 · Монетизация  ┼─ параллельно, после Фундамента
            → Фаза 5 · Корп-мета/эндгейм  ┘
```

**Первый спринт:** `FND-1` → `FND-7` (добавить источник credits + залочить паритет), затем `FND-2/3` → `FND-4`.

## Фаза 0 · Фундамент — свести прототип↔ядро + починить экономику *(сначала и в одиночку)*

Обкатанный контент (сбалансированная экономика, богатая наземка, эскадрильи) живёт в
прототипе, а авторитетный сервер гоняет расходящийся канон. **Замерено:** у credits на
каноническом пути нет петли дохода — стартовые 300 монотонно текут (−18/день на
2 крейсерах), ни `baseOutput`, ни здания-производителя, ни налога; движок при этом жив
(шахта даёт +480 metal/день). Фаза сводит расхождение и добавляет недостающий источник.

- **FND-1** · `[data]` · fix-unify · S · deps — → **Готово:** каждый `planetType` в `data/planetTypes.json` несёт `baseOutput` по ECON-7-константам прототипа; тест ядра проверяет, что terran-мир без зданий даёт владельцу >0 credits И >0 food/energy за спан. *Зачем:* добавляет **источник** credits, которого нет на сервере (`planetType.ts` уже читает поле — чистый порт данных, «включает» онлайн-экономику).
- **FND-2** · `[core]` · fix-unify · M · deps FND-1 → **Готово:** `tax.ts` экспортирует `taxModule` + `civicTax(n)=20/(1+0.06·(n−1))`; обитаемый мир добавляет credits через хук `economy.production`, империя из N миров даёт сублинейный итог; зарегистрирован в порядке сервера; `tax.test.ts` зелёный. *Зачем:* второй источник credits, на который опирается обкатанная экономика прототипа и которого ядру не хватает.
- **FND-3** · `[data]` · fix-unify · M · deps FND-1 → **Готово:** цены юнитов/зданий сведены к обкатанному прототипу (крейсер 220m→60m+20c и т.п.); канонический `buildings.json` получает недостающие экономические здания (refinery/farm/tax_office); ни один юнит/здание не ссылается на ресурс без источника. *Зачем:* канон сбалансирован под другую экономику (цены в 3–4× расходятся) — до паритет-теста надо свести к одному набору.
- **FND-4** · `[core]` · new-system · L · deps FND-3 → **Готово:** наземная фаза боя резолвится матрицей типов + `COMBAT_WIDTH` из прототипа вместо плоской attack/defense/hp-линии; ростер (militia/heavy_infantry/special_forces/tank) с пер-цель таблицами; орбитальная фаза не тронута; тесты на tank>infantry / спецназ грызёт броню / резерв ширины. *Зачем:* две наземные модели — живой форк; якорь 4 указывает на матрицу прототипа как каноническую (здесь концентрируется идентичность при плоских фракциях/гео).
- **FND-5** · `[data]` · fix-unify · M · deps FND-4 → **Готово:** `factions.json` = симметричный набор прототипа (общий ростер, различие только именем/косметикой + один мелкий плоский пассив ≤~0.12, без уникальных юнитов и фракционных зданий); каждый юнит роздан всем. *Зачем:* канонические vanguard/swarm асимметричны (у swarm biomass_pit, свой лоадаут) — прямое противоречие якорю 6. Убирает третий форк ростера.
- **FND-6** · `[proto]` · fix-unify · M · deps FND-5 → **Готово:** прототип строит матч из `data/maps/*.json` по `MatchMapSchema` (тот же формат, что у avaMapPool сервера), а не из хардкодного `MAP`; skirmish-1 и duel грузятся и играются headless. *Зачем:* три источника карт → один формат размера матча (якорь 5); карта, написанная раз, играется одинаково в прототипе и онлайн.
- **FND-7** · `[core]` · new-system · M · deps FND-1 → **Готово:** `economy-parity.test.ts` (ECON-5) прогоняет канон через балансные утверждения, которые держатся и для обкатанных чисел прототипа: окупаемость шахты L1 за 3–8ч, каждый обитаемый мир даёт >0 credits, ни один мир не даёт ноль. *Зачем:* постоянный гейт против повторного дрейфа прототип↔ядро.

## Фаза 1 · Глубина боя — combined-arms *(здесь игра перестаёт быть плоской)*

- **CA-1** · `[core]` · new-system · M · deps — → **Готово:** авианосец с эскадрильями на борту исполняет `fleet.launchWing` → новый быстрый флот-крыло в узле. *Зачем:* промоут эскадрилий из прототипа в ядро — keystone counter-web.
- **CA-2** · `[core]` · new-system · M · deps CA-1 → **Готово:** у крыла детерминированный счётчик вылетов (топливо ↓ на вылет; 0 → перезарядка); `fleet.recover` доккает крыло в со-локальный авианосец на дозаправку. *Зачем:* петля вылет↔возврат.
- **CA-3** · `[core]` · new-system · M · deps CA-1 → **Готово:** флот с `aaDamage`-кораблями бьёт по вражьим squadron-флотам в своём узле/орбите (зеркалит планетарное ПВО). *Зачем:* мобильное ПВО — противовес крыльям.
- **CA-4** · `[data]` · new-content · M · deps CA-3 → **Готово:** корабль `flak_escort` (мобильный `aaDamage`, слаб по корпусу) в данных; статы разведены так, что петля читается (истр > капиталы, ПВО/эскорт > истр, капиталы > эскорт). *Зачем:* делает RPS состава легибельным.
- **CA-5** · `[core]` · new-system · M · deps — → **Готово:** флот, бомбардирующий мир до штурма, копит ограниченное затухающее «размягчение» гарнизона, поднимая эффективную мощь атакующего в наземной фазе (§7.4). *Зачем:* настоящие комбинированные рода — космос кормит землю.
- **CA-6** · `[core][data]` · new-content · L · deps CA-1,CA-2 → **Готово:** крыло `repair_drone` восстанавливает КОРПУС (не щит) своим/союзным флотам в радиусе; единственный ин-бой хил. *Зачем:* саппорт-вариант эскадрильи.
- **CA-7** · `[core][data]` · new-system · L · deps — → **Готово:** схема парсит ракету (боеголовка, мало hp, скорость, сигнатура) + стат `missileCapacity` пусковой + `pointDefense` на юнитах/зданиях; неуправляемый полёт детектится радаром. *Зачем:* добавляет ракетный род в сеть контр.
- **CA-8** · `[core]` · new-system · M · deps CA-7 → **Готово:** на прибытии терминальное `pointDefense` (гарнизон+здания+корабельное из CA-3) бьёт по hp ракеты; выжившая детонирует. *Зачем:* ПВО-контра ракетам замыкает узел.

## Фаза 2 · Тех-дерево как раздатчик контента *(качественные анлоки = идентичность)*

- **TTQ-FND-1** · `[core][data]` · new-system · M · deps — → **Готово:** `TechnologyUnlocksSchema` получает `modules: string[]`; модуль корабля, чей id в `unlocks.modules`, отвергается билд-путём до открытия теха. *Зачем:* сеам для тех-гейтед лоадаута.
- **TTQ-EFX-1** · `[core][data]` · new-system · M · deps — → **Готово:** `TechnologyEffectsSchema` получает ≥2 качественных арм-ключа (напр. `artilleryRangeBonus`, `aaDamageBonus`), НЕ глобальный плоский урон, с проводкой в хуки соответствующего рода. *Зачем:* рост словаря эффектов под capstone-ветки.
- **TTQ-SQD-1** · `[data]` · fix-unify · M · deps — → **Готово:** `flight_decks` переезжает с `effects:{fleetSpeedBonus}` на `unlocks:{units:[strike_carrier, fighter_squadron]}` + перехватчик и элитный бомбер — авиа-род больше не строится с первого хода. *Зачем:* ветка эскадрилий раздаёт авиа-род.
- **TTQ-MSL-1** · `[core][data]` · new-content · L · deps TTQ-EFX-1 → **Готово:** `guidance_arrays` открывает `missile_frigate` (переиспользует трейт artillery/strikeRange, без правки кернела) + ПВО-контру; ракетная ветка их раздаёт. *Зачем:* ракетный род как контент дерева.
- **TTQ-GRD-1** · `[data]` · new-content · M · deps — → **Готово:** `combined_arms` перестаёт быть +5% и открывает наземный юнит (штурм-шагоход против укреплённого мира); `planetary_bastions` — капстоун-бастион. *Зачем:* наземная ветка раздаёт наземные рода.
- **TTQ-SPCMD-1** · `[data]` · new-content · M · deps — → **Готово:** `void_armadas` (поздний day-gate) открывает капитал `dreadnought` вместо +6%; `deep_survey` — сенсор-релей; логистические здания. *Зачем:* космос/командование раздают контент, не числа.
- **TTQ-MOD-1** · `[data]` · new-content · M · deps TTQ-FND-1 → **Готово:** `modules.json` получает арм-флейворные модули под правило горизонталь/вертикаль (флак-`defense`, ракетный launcher-модуль), гейт через `unlocks.modules`. *Зачем:* лоадаут — рычаг идентичности.
- **TTQ-HERO-1** · `[data]` · new-content · M · deps — → **Готово:** ≥3 новых геройских фиттинга/абилки в уже-живые сеамы (пассив на `fleet.speed`/`combat.damage` или абилка counter-web). *Зачем:* герои — рычаг идентичности при плоских фракциях.
- **TTQ-PACE-1** · `[data]` · fix-unify · M · deps TTQ-SQD-1,TTQ-MSL-1,TTQ-GRD-1,TTQ-SPCMD-1 → **Готово:** после контент-кирпичей остаточные чисто-% узлы урезаны до малого явного экономического бюджета (с задокументированным «почему остаётся числом»); day-gate эпох под оба темпа. *Зачем:* ребаланс дерева под быстрый и медленный режим.

## Фаза 3 · Ретеншн-примитивы *(оба темпа — без них медленный режим мёртв)*

- **RET-1** · `[core][data]` · new-system · M · deps — → **Готово:** `comebackModule` читает `match.scores` и через хук `economy.production` даёт отстающим бонус к производству по разнице в счёте. *Зачем:* анти-черн для проигрывающих (самый мелкий выигрыш — первым).
- **RET-2** · `[core][data]` · new-content · M · deps — → **Готово:** когда место переходит от пусто/ИИ к человеку ПОСЛЕ прогресса часов (Iron Order entry window), ядро эмитит разовый стартовый грант. *Зачем:* поздний вход не «мёртв по прибытии».
- **RET-3** · `[srv]` · new-system · M · deps — → **Готово:** `AccountStore` получает CRUD Web Push подписок (Memory+Postgres, дедуп) + VAPID-конфиг + шов `PushSender`; `POST /push/subscribe`. *Зачем:* инфра под несущий ре-энгейджмент.
- **RET-4** · `[cli]` · new-system · M · deps RET-3 → **Готово:** service worker (`packages/client/public/sw.js`) ловит `push` → уведомление с дип-линком по клику. *Зачем:* доставка уведомления игроку.
- **RET-5** · `[srv]` · new-system · M · deps RET-3 → **Готово:** поток `events` комнаты мапится на ретеншн-события (флот прибыл / атакован / исследование готово) → push подписчикам. *Зачем:* триггеры «пока тебя не было».
- **RET-6** · `[srv]` · new-system · M · deps — → **Готово:** `buildRecap` (уже DOM/i18n-free) промоутится в общее место, сервер собирает офлайн-дайджест на реконнект. *Зачем:* догоняющая сводка (ONB-5).
- **RET-7** · `[core]` · new-system · L · deps — → **Готово:** `orderChainModule` — ограниченная per-fleet цепочка в `state.orders` (JSONB-safe, шаги move/orbit/assault, лимит длины), исполняется драйвером офлайн. *Зачем:* возвращает фантазию «выставил приказы → проснулся с результатом» (вырезана в REL-1).
- **RET-8** · `[srv][core]` · new-system · M · deps RET-3,RET-5 → **Готово:** `rally.set {at, targetNode, allies}` — таймер скоординированной атаки; на срабатывании ядро пушит союзникам. *Зачем:* соц-обязательство — самый прочный крючок жанра.

## Фаза 5 · Корп-соц мета + эндгейм + dual-tempo

- **TEMPO-1** · `[srv]` · fix-unify · M · deps — → **Готово:** справочник `TEMPO_CLASSES = { fast:{timeScale:24,…}, slow:{timeScale∈1..4,…} }`; `MatchKeeper` сеет матчи выбранного класса. *Зачем:* оба темпа как выбираемые (якорь 1).
- **META-LEDGER-1** · `[srv]` · new-system · L · deps — → **Готово:** на `match.ended` сервер идемпотентно кредитует каждому севшему аккаунту xp/сессии в durable леджер. *Зачем:* прочный источник кросс-матчевого прогресса.
- **CORP-CHAT-1** · `[srv]` · new-system · L · deps — → **Готово:** `CorpChatStore` + relay: член шлёт `corpChat.send` → сервер режет текст, проверяет членство (не-член → `E_FORBIDDEN`), rate-limit, bounded backlog. *Зачем:* постоянный корп-чат — соц-склейка первого мета-столпа.
- **CORP-PRESENCE-1** · `[srv]` · new-system · M · deps — → **Готово:** присутствие членов в `EphemeralStore` с TTL (хартбит от сессии) → online/in-match/offline. *Зачем:* «кто сейчас в игре».

## Заметки и открытые нюансы

- **Credits (уточнено замером):** канон не «сломан целиком» — движок жив (шахта даёт metal); конкретно у **credits нет петли дохода** (−18/день сток от старта). Рынок лишь перераспределяет credits между игроками (+5% сток), не создаёт доход. `FND-1`+`FND-2` добавляют источник.
- **Каскад идентичности:** при плоских фракциях/гео глубина держится на тех/составе/героях/лоадауте — если они окажутся тонкими, игре нечем различать билды (следить в Фазах 1–2).
- **Непокупаемая вертикаль:** её потолок надо задать так, чтобы «престиж мастерства», а не «+X% каждый сезон» (`PROG-4` фиксирует бюджет).
- **Соло без корпы:** первый мета-столп — корп-клуб, он помогает членам корпораций; ладдер/сезоны (кросс-матчевый крючок для соло) — следующий столп, не в этой пачке.
