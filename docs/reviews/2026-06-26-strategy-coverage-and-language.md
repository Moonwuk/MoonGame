## 3. Язык: TypeScript — верно?

**Да. Ядро на TS, один язык сервер+клиент — оставить.** Вывод устоял против «адвоката переписывания» при трёх малых in-TS правках (все сделаны, §4).

**Почему верно (структурно):** центральная ставка — authority-сервер + бит-в-бит превью в браузере, сверяемые `hashState` — имеет нулевую «поверхность перевода» только в JS/TS. Нативный сервер требует второго клиента и молитвы о бит-идентичности; WASM-мост тащит JS↔WASM-границу на превью-путь ради проблемы, которой у кода нет.

**Детерминизм (верифицировано по спеке):**
- `+ − × ÷ √ min max floor ceil` + целочисленные — **корректно округляются, бит-идентичны** во всех IEEE-754-движках.
- **`Math.sqrt` бит-идентичен ПО СПЕЦИФИКАЦИИ:** IEEE-754 предписывает корректное округление `√`, и ECMA-262 **не** относит `Math.sqrt` к списку «implementation-approximated» (где `sin/cos/exp/log/pow/atan/hypot`). Сильнейший довод скептика («sqrt разный на движках») — **опровергнут**. Живой `route.ts:16 Math.sqrt` безопасен.
- **`Math.hypot` — единственная негарантированная** операция; была только в `buildFromMap.ts` (билд-тайм, не редьюсер) — **заменена на `sqrt`** в этом заходе.
- В `shared-core` трансцендентных нет. (`server/scenario.ts:147-148` `Math.cos/sin` — server-side авторинг, `Math.round` в целые, вне детерминированной границы — безвредно.)
- **Найдено и исправлено:** `architecture.md:496` утверждал «float-расхождений by design нет» — **ложно**; детерминизм *удержан* подмножеством, не «невозможен by design».

**Где другой язык уместен — с триггерами:**

| Поверхность | Язык | Решение | Триггер |
|---|---|---|---|
| Персистентность | Postgres (JSONB) | **Делать** | Этап 3. `GameState` уже JSON → в JSONB без маршалинга. Без тяжёлой ORM. |
| Sim-ядро | Rust→WASM | **Отложить** | Только при **И** замороженной схеме **И** *измеренном* сбое (воспроизведённый desync на бит-точном подмножестве — крайне маловероятно — **или** упёртый CPU редьюсера). Трансцендентные сперва — fixed-point+LUT в TS. |
| WS-шлюз (край) | Elixir/Go | **Отложить** | Тысячи живых матчей, кластер Node — *измеренное* узкое место. Только транспорт, не sim. До этого — cluster-mode Node + Redis fan-out. |
| Аналитика/античит-ML | Python | **Отложить** | Пост-лонч. Первый слой античита — replay на TS-ядре (Python не нужен), Python — второй (статистический). |
| Нативный мобайл | Kotlin/Swift | **Никогда** | PWA-first покрывает жанр; максимум — тонкая обёртка вокруг PWA. |

**Итог:** TS верен; единственная «живая» опасность (`Math.sqrt`) бит-точна по спеке; машинерия детекта теперь подключена; переписывание сменило бы удержанный риск на неудержанный на golden-границе нового языка.

## 4. Применено в этом заходе (✅ Fixed)

Из этого ревью на `main` сведена детерминизм-дисциплина:

1. ✅ **`architecture.md` §8** — ложное «float-расхождений by design нет» → «риск удержан бит-точным подмножеством IEEE-754; остаток ловит `hashState`».
2. ✅ **ESLint-гард** (`eslint.config.js`) — `no-restricted-properties` в `shared-core/src/**` (не-тест) банит 22 implementation-approximated `Math`-функции (`hypot/sin/cos/tan/exp/log/pow/cbrt/atan/…`); `Math.sqrt` намеренно разрешён. `buildFromMap.ts` `Math.hypot` → `Math.sqrt`-идиом (как в `route.ts`).
3. ✅ **`hashState`-десинк** — на `main` уже подключён **независимо** через серверный опц. `emitStateHash` (`HashField` на снапшотах `welcome`/`state`/`delta`; клиент сверяет хеш реконструкции). Поэтому из этого ревью обвязка не дублировалась — взяты только правки 1–2. _Дальше: авто-resync при mismatch + сравнение с предсказанием превью (это превращает любой будущий «языковой» вопрос в измерение)._
